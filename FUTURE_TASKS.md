# Future Tasks

Feature ideas for Terminal Trials, roughly ordered by impact.

## Gameplay Modes

- **Timed Challenge Mode** — Speed-run a full zone against a timer. Local leaderboard for best times. Creates replayability for users who've cleared everything.

- **Combo/Chain Challenges** — Multi-step challenges where each step builds on the last (e.g., find .log files → count lines → redirect to report). Tests real workflow thinking.

- **Daily Challenge / Random Encounters** — A single rotating challenge each session from a bonus pool. Keeps people coming back after clearing all zones.

## Feedback & Learning

- **Better Failure Feedback** — When a user gets an answer wrong, show *why* their answer wouldn't work, not just the correct answer (e.g., "You typed `ls -a` but the challenge needed long format too").

- **Progressive Codex Unlocking** — Codex entries unlock as users clear zones instead of being available from the start. Makes the codex feel earned.

## Polish & Engagement

- **ASCII Art / Narrative** — Lean into the RPG framing: mini-stories per zone, a "boss challenge" at the end of each zone (harder multi-part problem), NPC guide character.

- **Achievement / Badge System** — Specific achievements beyond ranks: "Completed a zone with no hints", "Perfect flash drill 3x in a row", "Cleared all zones". Visual badges in the stats screen.

- **Challenge Sharing** — Export a challenge as a shareable code that someone else can import and try. Low-fi multiplayer without networking.

- **Terminal Bell Sound Effects** — Use `\x07` (BEL) sparingly for correct/wrong feedback, or terminal-compatible sounds where available.
